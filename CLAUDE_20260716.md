## ⚠️ STANDING CODE STANDARDS — ENFORCED EVERY SESSION

### DUB C 2.0 — T-SQL Style (AUTO-APPLY TO ALL SQL OUTPUT)

These rules are non-negotiable and apply to **every** T-SQL block produced in any session:

| Rule | Requirement |
|------|-------------|
| **Comments** | Preserve ALL comments from original code |
| **No alignment spacing** | Zero extra whitespace for alignment — one space max between any two tokens |
| **Comma placement** | Comma on LEFT side of each field, NO preceding spaces: `,fieldName AS [Alias]` |
| **JOIN fields** | ON clause stays on same line as JOIN |
| **Explicit JOIN type** | Always write `INNER JOIN` (never bare `JOIN`). Never create `LEFT INNER JOIN` or `RIGHT INNER JOIN` |
| **Do NOT change** | `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`, `CROSS JOIN`, `CROSS APPLY`, `OUTER APPLY` — leave as-is |
| **SELECT** | Never leave SELECT alone on its own line |
| **AS keyword** | All table AND field aliases preceded by `AS` |
| **UPPERCASE** | All SQL syntax keywords in UPPERCASE |
| **CASE structure** | `CASE`, `WHEN`, `THEN`, `ELSE`, `END` each on their own new line |
| **DISTINCT → GROUP BY** | Replace `DISTINCT` in final SELECT with explicit `GROUP BY` for optimizer clarity |
| **UNION → UNION ALL** | Prefer `UNION ALL` when duplicates are handled in final aggregation |
| **INSERT safety** | Every inserted field gets explicit `CAST` matching the `CREATE TABLE` data types exactly |
| **Index consideration** | Always evaluate whether an index would benefit the query |
| **FROM placement** | `FROM` keyword starts at beginning of line; base table on same line as `FROM` |
| **JOIN indentation** | One TAB per structural join level; sibling joins same level; child joins +1 tab |
| **No space-alignment** | Never use spaces to align JOIN keywords, table names, aliases, ON conditions, or comments |
| **(NOLOCK)** | Applied directly after table name, before alias: `TableName (NOLOCK) AS alias` |
| **Production ready** | Always deliver complete, production-ready code |

**CORRECT examples:**
```sql
-- ✅ Comma left, no alignment spacing
,1 AS [QtrNum]
,DATEPART(YEAR,@ClockStart) AS [Year]

-- ✅ JOIN structure
FROM HMOPROD_PLANDATA.dbo.enrollkeys (NOLOCK) AS ek
CROSS JOIN INFORMATICS.dbo.date_calendarISO (NOLOCK) AS dc
	INNER JOIN INFORMATICS.dbo.[uvw_LINE_OF_BUSINESS] (NOLOCK) AS bp ON ek.programid = bp.programid
		LEFT JOIN HMOPROD_PLANDATA.dbo.language (NOLOCK) AS lang ON lang.languageid = mem.primarylanguage

-- ✅ Column definition spacing
[DaysRemainingMonth] [int] NULL,
```

**INCORRECT examples:**
```sql
-- ❌ Alignment spacing
       ,1 AS [QtrNum]
       ,DATEPART(YEAR,@ClockStart) AS [Year]

-- ❌ Column definition with alignment spacing
[DaysRemainingMonth]    [int]        NULL,
```
