Runbook Sequence — NCQA_UM_20260610.sql
01 · Set Parameters

Verify and set all runtime parameters: @ClockStart = '06/14/2025', @ClockStop = '05/15/2026', @BRAND = 'CHGSD', @Authlob = 'MEDI-CAL', @PCPrecordRANK = 1. Confirm date range aligns with the current NCQA accreditation window per submission narrative. Never run with @Authlob = NULL or @BRAND = NULL in production.

02 · Drop / Create Permanent Tables

The script drops and recreates INFORMATICS.dbo.NCQA_UM_DENIALS and INFORMATICS.dbo.NCQA_UM_DENIALS_OUTPUT at the top of each run. Confirm INFORMATICS.dbo write permissions for CHG\wcarr on SQLPRODAPP01 are active before executing.

03 · Pre-Filter Anchor (#QUALIFYING_AUTHS)

Step 01 builds #QUALIFYING_AUTHS. Immediately check SELECT COUNT(*) FROM #QUALIFYING_AUTHS. Expected: several hundred thousand rows for a full accreditation window. Zero rows = parameter error or no matching notification letters. Do not proceed if count is zero.

04 · setauthref CTE + Main INSERT (Steps 02–03)

The setauthref CTE reads from #QUALIFYING_AUTHS and applies the LOB/brand gate. Main INSERT to NCQA_UM_DENIALS runs with 15+ JOINs. Check row count in NCQA_UM_DENIALS after INSERT. CREATE INDEX idx_REFACTOR_DupID runs immediately after INSERT — confirm it completes before proceeding.

05 · Enrichment UPDATEs (Steps 04–07)

Run all UPDATE blocks sequentially: AOR, Dx (STRING_AGG), CPT (STRING_AGG), RevCode (STRING_AGG), Part B drug (PartB CTE), #authattributes single-pass build, 4 attribute UPDATEs, TAT C010 ClockStop override, Threshold/Units, CROSS APPLY business-day TAT calculation.

06 · PAR Status Chain (Step 08)

Three-tier PAR lookup runs twice (ReferFrom then ReferTo). Confirm uvw_FALSE_PROVIDER_PAR, uvw_PROVISO, and SQLPRODAPP01.INFORMATICS.dbo.uvw_ProviderRepository274_PAR are all accessible. The cross-server view requires an active linked server connection to SQLPRODAPP01 from SQLPROD02.

07 · Final Assembly (Step 09)

Complex GROUP BY SELECT INTO NCQA_UM_DENIALS_OUTPUT joining PCP (uvw_PCP), Aid Code (EnrollmentManager), language, and MBI. ROW_NUMBER() deduplication applied. Verify row counts before and after to confirm expected dedup ratio.

08 · NCQA Final SELECT (Step 10)

refertoreferfrom CTE + final SELECT producing 9-column NCQA output (Columns A–I). Verify: all 4 file types present (McaidPS/CU/PU/PN), no NULL values in any NCQA column, TAT values are within expected ranges per file type. Export result set to Excel for NCQA submission.

09 · Post-Run Validation

Cross-reference row counts between NCQA_UM_DENIALS (Step 03 universe) and final SELECT output (Step 10). Investigate any significant drop. Check TAT distribution: flag negative TAT or TAT > 90 days as data quality issues. Confirm @ClockStart and @ClockStop dates appear correctly in Column D and Column H values.

10 · Delivery

Export final SELECT result set to Excel. Deliver per NCQA accreditation coordinator instructions. File naming convention: NCQA_UM_MEDI-CAL_CHGSD_[YYYYMMDD].xlsx. Retain NCQA_UM_DENIALS and NCQA_UM_DENIALS_OUTPUT in INFORMATICS.dbo until NCQA review is complete — do not drop manually.

Quality Controls & Business Rules
Control ID Control Name Rule / Check Failure Impact Step
C001 LOB Lock @Authlob must = 'MEDI-CAL'. Enforced via LINE_OF_BUSINESS LIKE '%'+@Authlob+'%' in setauthref CTE. NULL returns all LOBs. Audit universe inflated with non-Medi-Cal authorizations. NCQA submission scope violation. 02
C002 Brand Lock @BRAND must = 'CHGSD'. Enforced via bp.[Contract Number] = ISNULL(@BRAND, ...) in setauthref CTE. NULL returns all brands. CHGLA/other brand records contaminate CHGSD submission. File rejection risk. 02
C003 Status Filter r.status = 'DENIED' enforced at #QUALIFYING_AUTHS (Step 01) and again in NCQA Final SELECT (Step 10). Non-denied authorizations (APPROVED, PENDED, etc.) enter the NCQA universe. Submission error. 01, 10
C004 Notification Letter Date in Window TRY_CONVERT(date, attrib.effdate) BETWEEN @ClockStart AND @ClockStop. The pre-filter anchor ensures only auths notified in the NCQA accreditation window are included. Auths outside the accreditation window enter the universe. TAT values may be out-of-scope. 01
C005 ClockStop C010 Override After #authattributes is built, UPDATE sets ClockStop = notification letter date. Without this override, TAT calculates to internal decision date, not member notification date — incorrect per NCQA narrative. TAT values do not align with NCQA requirements. Column I incorrect for all records. 07
C006 File Type Coverage All 4 file types (McaidPS, McaidCU, McaidPU, McaidPN) must appear in Column C. SELECT COUNT(*) FROM ... GROUP BY [File Type] to verify. No NULL File Type values permitted. Missing file type = incomplete NCQA submission. NULL in Column C = submission rejection. 10
C007 TAT Reasonableness Post-service (PS) ≤ 30 days. Concurrent (CU) ≤ 1 business day. Pre-service urgent (PU) ≤ 3 business days. Pre-service standard (PN) ≤ 14 calendar days. Flag negatives and >90 days. Outlier TAT values indicate data quality issues — notification letter date may be wrong or missing. 07, 10
C008 Permanent Table Write Permission INFORMATICS.dbo.NCQA_UM_DENIALS and NCQA_UM_DENIALS_OUTPUT require CREATE TABLE / DROP TABLE / INSERT permissions for CHG\wcarr on SQLPRODAPP01. Verify before run. Script fails at Step 03 INSERT if permissions are not current. No output produced. 03, 09
C009 Cross-Server PAR Connectivity SQLPRODAPP01.INFORMATICS.dbo.uvw_ProviderRepository274_PAR requires active linked server from SQLPROD02 to SQLPRODAPP01. Verify with SELECT TOP 1 FROM the view before full run. PAR lookup chain fails at Tier 3. ReferFromPAR and ReferToPAR may be NULL for all records. 08
C010 UTC → Pacific Standard Time Conversion r.UtcReceiptDate must be converted to Pacific Standard Time using AT TIME ZONE pattern. Bare GETDATE() or no conversion produces incorrect ClockStart and TAT values. TAT calculation is off by UTC offset (typically 7–8 hours). May affect same-day determination logic. 03, 07

Performance Notes: NCQA_UM_20260610.sql incorporates two major optimizations over prior versions. (1) #QUALIFYING_AUTHS pre-filter (Step 01): anchors the working set on notification letter date + DENIED status before any LOB or enrollment joins — dramatically reduces the rows entering the 15+ JOIN main SELECT. (2) #authattributes single-pass (OPT002, Step 06): one read of qattribute + referralattribute replacing five separate reads for MODIFIED / EXTENSION / NOTIFICATION / THRESHOLD / ORAL attribute types. Both optimizations were user-applied to address excessive runtime observed in prior execution plans (NCQA CTE STAGING.sqlplan / NCQA CTE temp TABLE.sqlplan).
