Stored Procedure Parameters — sp_IHA_MEDI_CAL
All parameters are declared at the top of the SP and control report scope. Any change must be documented in the SP COMMENT(S) / CHANGE.LOG section with a C### reference.
@ClockStart
date = '01/01/2020'
Report start date. Set per Ms. Gabby email 20211215. Anchors the enrollment and IHA lookback window.
@ClockStop
date = GETDATE()
Report end date. Dynamic — resolves to today's date at execution. Ensures report always reflects current-day eligibility cutoff.
@lob
nvarchar(255) = 'MEDI-CAL'
Line of business filter applied to LINE_OF_BUSINESS via LIKE. Valid options: MEDI-CAL, DSNP MEDI-CAL PLAN, COMMUNITY Y MAS, etc. NULL = all LOBs.
@ContinuousMonths
decimal(9,0) = 3
Minimum consecutive months of enrollment required for a member to qualify as eligible for IHA. Do not reduce below 3.
@AgeCutoff
decimal(9,0) = 20
Members must be age ≥ 20 to be included in the IHA eligible population. Targets adult Medi-Cal members.
@GAPOF / @MAXGAP
int = 1 / int = 1
Enrollment gap tolerance parameters. A gap of 1 month is allowable; gaps exceeding @MAXGAP=1 break the continuous enrollment window.
@lag / @lagyrs / @lagdays
int = 0 / 0 / 0
Date offset parameters for @when calculation. Currently all zero — no lag applied. Adjust +/- to shift the effective period reference date if needed for retrospective analysis.
@PCPrecordRANK
int = 1
Controls PCP attribution rank. SET = 1 returns only the most current PCP record per member via DENSE_RANK() OVER (PARTITION BY memid ORDER BY effdate DESC).
@CCIIndicator
varchar(4) = DATEPART(yyyy,…)
CCI year indicator derived from @lag-adjusted GETDATE(). Per Ms. Maryanne 834. Dynamic year unless overridden with a hard-coded year value.
@isoday
int = 1
Calendar ISO day — controls which day of the month is used as the effective date for eligibility determination. Default = 1 (first of month).
@isomem
nvarchar(25) = NULL
Sample/test member ID filter. NULL = all members. Set to a specific memid (e.g., '%1083184%') for targeted QA of an individual member's IHA record.
@ExecDate
datetime = GETDATE()
Execution timestamp captured at run time. Surfaced in the SSRS email RESULT header row to document when the report was generated.
