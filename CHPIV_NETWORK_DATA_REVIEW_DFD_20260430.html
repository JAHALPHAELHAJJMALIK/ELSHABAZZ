Change Log — February 2026 → April 2026

REMOVE 'CHPIV Direct' providers from EXTERNAL file. Per Ms. Johanna conversation 20260116. Subject: "RE: CHPIV Network Data Alignment." WHERE NOT IN ('CHPIV Direct'). Both builds.

Full new section: six temp tables (#chpiv, #hcai, #LIC_FAC, #LIC_PROV, #PHONEprimary, #baselineprovider, #baselinefacility) plus INSERT from chpivdirectnonprovdir alias. April only.

APRIL [Data Sourced From] Rename

PROVDIR legs: 'CHGSD CHPIV Direct Contract' (Feb) → 'CHPIV Direct Provider Directory' (April). New label: 'CHPIV Direct Non - Provider Directory' for eVIPS direct leg. April only.

APRIL NOT EXISTS NPI Gate

Explicit anti-join on NPI_NO in both #baselineprovider and #baselinefacility guards against double-counting eVIPS records that already exist from PROVDIR legs. April only.

APRIL HCAI ID Infrastructure

#hcai temp table from vwLocationServices WHERE ServiceTypeName LIKE '%HCAI%'. Implements DMHC MY204 / APL 24-019 HCAI ID requirement for Hospital and Clinic Report Form. April only.

APRIL Taxonomy CTE (Commented)

chpivnpitax CTE ready but commented. Joins tblNPIData via CROSS APPLY across 15 taxonomy code columns → ANCTaxonomyCrosswalk. Infrastructure for future TAXONOMY CODE population. April only.

File size delta: February ~51KB → April ~126KB (+148%). The size increase is almost entirely attributable to the 7-column add-on schema propagated across every UNION leg (each leg adds ~700 characters of repeated column definitions) plus the new Section 3 eVIPS block (~50KB of complex join logic).
Analyst Controls / Quality Gates
ControlRecommended validationFailure symptom / risk
C001 — GROUP exclusion Confirm WHERE NOT IN ('CHPIV Direct') is present in the CHPIV_PROVIDER_DIRECTORY_DATA seed SELECT before any execution. CHPIV Direct providers flow in via the External file and duplicate records already inserted from the Direct contract arm.
CHPIV-DSNP network filter Confirm all PROVDIR and eVIPS UNION legs include AND [Participating Network] IN ('CHPIV-DSNP') or ProductName IN ('CHPIV-DSNP'). Non-CHPIV provider records (CHGSD, other networks) contaminate the CHPIV network data table.
eVIPS archive / status filters Confirm all six archive flag AND conditions and PractitionerStatus / PracticeStatus / StatusTypeName conditions are intact in both #baselineprovider and #baselinefacility. Inactive, terminated, or archived providers/facilities appear in the network directory output.
NOT EXISTS NPI dedup gate Verify Section 2 COMMIT has fully executed before Section 3 begins. Inspect NOT EXISTS subquery in both baseline temp tables. If TRY block rolls back Section 2, Section 3's NOT EXISTS check passes incorrectly and creates duplicates on re-run.
IDENTITY PK integrity Confirm DROP COLUMN ID + ADD ID IDENTITY(1,1) NOT NULL PRIMARY KEY executes before the Section 2 INSERT. PK violation or missing IDENTITY column breaks downstream system joins that depend on the ID surrogate key.
Post-insert GROUP / COUNTY audit Review GROUP distribution — confirm CHPIV-DSNP is the dominant or sole group. Review COUNTY — all CHPIV records should reflect Imperial County per business rule. Unexpected GROUP values suggest a filter regression. Non-Imperial COUNTY values suggest a ZIP_CODES crosswalk miss or a provider located outside CHPIV service area.
7 add-on column population Confirm [TAXONOMY CODE] intent (decimal(10,0) — ⚠ KEY ANCHOR per inline comment). Activate chpivnpitax CTE when taxonomy data is ready. Verify #hcai table is populated before HCAI ID is mapped. All 7 columns remain NULL/0 indefinitely if CTE is never activated, creating a schema gap for the DMHC MY204 / HCAI ID submission requirement.
[Data Sourced From] label accuracy Confirm three distinct labels in April output: 'EXTERNAL CHPIV ROSTER xlsx FILE', 'CHPIV Direct Provider Directory', 'CHPIV Direct Non - Provider Directory'. Run a GROUP BY [Data Sourced From] COUNT(*) to verify all three populate. Records from Section 3 appearing with Section 2's label (or vice versa) indicate a copy-paste error in the SELECT stub for a specific UNION leg.
Senior analyst note: The strongest near-term improvement opportunity is activating the TAXONOMY CODE and HCAI ID population paths currently stubbed at NULL/0. The infrastructure (chpivnpitax CTE, #hcai temp table) is fully written in the April build — only the ⚠ TAXONOMY CODE intent confirmation and the #hcai-to-[HCAI ID] mapping wiring remain outstanding to fulfill DMHC MY204 compliance for the CHPIV Hospital and Clinic Report Form.
