Source Tables Reference — All Tables Touched by DHCS_LTC Scripts
Five source databases contribute to this report. Temp tables are session-scoped. Permanent INFORMATICS tables are dropped and recreated each script execution. Brand suffix [DSNP] or [CHPIV] applies to all permanent output tables.
HMOPROD_PLANDATA · QNXT Transactional (SQLPROD02)
dbo.claim
Source
Claim header. Provides startdate, enddate, dischargedate, admitdate, planid, formtype, facilitycode, billclasscode, frequencycode, drg, patientstatus, admitsource, admithour, affiliationid, contractnetworkid, totalamt, totalpaid, status, controlnmb, logdate. Primary source in #QUALIFYING_CLAIMS and CLAIMDATA CTE.
dbo.claimdetail
Source
Claim line. Provides revcode (LTC gate), servcode, amountpaid, contractpaid, claimamt, servunits, modcodes, location, contractid, termid, ffspoolamt, ProvMedicareMandatedAdjust, Capitated, paydiscount, ineligibleamt, cobeligibleamt, extpaidamt. Revcode normalization handles leading-zero and Z-prefix formats.
dbo.claimdrg
Source · C001
MS-DRG (Medicare Severity DRGs). LEFT JOINed on claimid WHERE finaldrg+SeverityOfIllness IS NOT NULL. Produces [finalDRG+SOI], subdrg, moddrg, finaldrg columns. Added C001 20240523.
dbo.claimdiag
Source
Claim diagnosis codes. ROW_NUMBER() OVER (PARTITION BY claimid ORDER BY sequence ASC, codeid DESC) = 1 to isolate principal diagnosis. Provides codeid (ICD), diagtype, poa (Present on Admission), sequence. LEFT JOINed to DiagCode for description.
dbo.claimproc
Source
Claim procedure codes. Used in QUPD of [PROCEDURE_1_CODE]: WHERE proctype='Primary' AND claimline='1'. Provides pcode for the primary procedure code column.
dbo.ClaimValue
Source
UB value codes. Used in acccode CTE: WHERE valuecode='24' AND (amount=.75 Vent OR .76 Non-Vent). Provides [Accommodation Code] per Rodrigo Claims guidance. One value per claim.
dbo.affiliation
Source
The rendering/payto relationship table. affiliationid links claim to a.provid (Rendering Provider) and a.affiliateid (PayTo/Vendor). Key for AdmitID construction (memid+startdate+provid) and QA provider SELECTs.
dbo.provider (x2)
Source
Provider table joined twice: p1 (Rendering) on a.provid → NPIprovnm, PROVNM; p2 (PayTo) on a.affiliateid → NPIpayto, PAYTONM. NPI used in 274 PAR override UPDATE.
dbo.member
Source
Member demographics. Provides fullname (→ MEMNM), dob (→ MEMBER_AGE calculation). LEFT JOINed on CLAIMDATA.memid.
dbo.hcfaposlocation
Reference → #POSLOC
Place of service location codes. Pre-materialized as #POSLOC (clustered on locationcode) in COPT Step 02 to eliminate Nested Loop rebinds. Provides POS description for the location column.
dbo.contract
Reference → #CONTRACTDESCR
Contract descriptions. Pre-materialized as #CONTRACTDESCR (clustered on contractid) in COPT Step 02. Provides [CONTRACT DESCRIPTION] via UPPER(LTRIM(RTRIM(description))) WHERE not blank.
dbo.contractinfo
Source
Contract participation info. Five-key JOIN (programid + affiliationid + contractid + networkid + DOS in range) to derive contractedVerification and capitationVerification. Per Sir Adrian: affiliation sequence is preferred PAR method.
dbo.enrollkeys
Source
Master enrollment roster. Used in [Primary/Secondary Status] QUPD: JOIN on memid+enrollid → effdate, termdate, segtype='INT', DOS BETWEEN eff and term. Provides carriermemid, programid, planid.
dbo.enrollstatus
Source
Enrollment status. Provides primarystatus ('P'=primary, 'S'=secondary) for [Primary/Secondary Status] column and D-SNP QUANTITY CASE logic.
dbo.enrollcoverage
Source
Enrollment coverage. Provides ratecode. Joined in [Primary/Secondary Status] QUPD alongside enrollstatus for rate code context per active coverage segment.
dbo.svccode · DiagCode · revcode
Reference
Code description lookups: svccode → CPT Service Description (LEFT JOIN on servcode); DiagCode → diag descr (JOIN in claimdiag subquery); revcode → REVCDE Descr (normalized JOIN on revcode). All three return description strings for readable output columns.
QNXT_Custom.TRI.FfsPaymentDetails

Targeted Rate Increase payment amounts. JOIN on ClaimId + LineNum → ExtraAmountPaid. Used in QUPD Step 05 to populate [TRI Payment] column. Per Sir Evan email 20241202.

Runbook Sequence — DHCS LTC D-SNP Dual-Brand Scripts
01 · Set @BRAND and Confirm Quarterly Period

Determine execution target: CHGSD (DSNP) or CHPIV (MSO-CHPIV). Set @BRAND accordingly. Confirm @leadlag=-4 and @leadlagmonths=3 are appropriate for the target submission quarter. Run the parameter QA SELECT to verify @RangeStartDate/@RangeEndDate and Reporting Period string (e.g. '2026 Q1'). Confirm the quarterly submission due date has not passed.

02 · Verify uvw_LINE_OF_BUSINESS Returns Active Rows

The parameter QA SELECT joins uvw_LINE_OF_BUSINESS WHERE LINE_OF_BUSINESS LIKE @lobplanprog AND BRAND IN (@BRAND). Confirm rows are returned. Zero rows = LOB/BRAND mismatch — verify the plan names in uvw_LINE_OF_BUSINESS before proceeding.

03 · Execute Pre-Stage (COPT Steps 01–02)

DROP TABLE IF EXISTS for all output tables (#POSLOC, #CONTRACTDESCR, #QUALIFYING_CLAIMS, DHCS_LTC_[BRAND], DELIVERABLE_DHCS_LTC_[BRAND], DELIVERABLE_[BRAND]_TEMPLATE). Confirm pre-stage lookup tables build without error. Confirm #QUALIFYING_CLAIMS DUP validation returns 0 rows and QA COUNT SELECT shows LTC-classified claims (non-zero Unique Claims and Unique Members).

04 · Run Main SELECT INTO DHCS_LTC_[BRAND]

Execute the setclaim+CLAIMDATA CTE chain into the permanent table. Confirm DupID DUP validation returns 0 rows. Review QA [RANGE] SELECT for date min/max alignment with @RangeStartDate/@RangeEndDate. Review QA LOB(S) SELECT to confirm only the expected LINE_OF_BUSINESS is present.

05 · Execute 8 QUPD Passes

Run all sequential UPDATEs: THE FINAL CLAIM, Accommodation Code, TRI Payment, PROCEDURE_1_CODE, prindiag+POA, Primary/Secondary Status, contractedVerification (contractinfo JOIN + 274 PAR override), QUANTITY, HCT LOGIC DAY(S) (C003). Review QA CONTRACTED PROVIDER(S) SELECT to confirm expected LTC facilities appear as contracted. Verify [HCT LOGIC DAY(S)] is non-zero for LTC claims.

06 · Build #AdmitAssessment + WHILE Loop

Verify the CONSECUTIVE stay WHILE loop terminates (@@ROWCOUNT = 0 BREAK). Confirm #AdmitAssessment DUP validation on Cx+AdmitID returns 0 rows. Review sample DETAIL SELECT (WHERE memid IN ('...')) for a known LTC member to confirm [Index Admission Date], [Index Discharge Date], and [HCT LOGIC DAY(S)] are populated correctly.

07 · Build DELIVERABLE + TEMPLATE

Confirm DELIVERABLE_DHCS_LTC_[BRAND] contains expected members (DAY(S) >= 90, discharge date in range). Review SUMMARY BY RENDERING SELECT for top LTC facilities. Run SELECT * FROM DELIVERABLE_DHCS_LTC_[BRAND]_TEMPLATE to verify the 5-column DHCS format: Parent Company Name = 'Community Health Group', H Contract # = 'H4377/D-SNP', Reporting Period = correct quarter, EAE or FIDE = 'EAE', member count is within expected range.

08 · Repeat for Second Brand Then Submit

Execute the complementary brand script (CHGSD if CHPIV was run first, or vice versa) following steps 01–07. Compare member counts between brands for reasonableness. Share DELIVERABLE_TEMPLATE results with Ms. Barbara Vargas, RN, MSN for DHCS quarterly submission. Document execution date and reporting period in the change log.

Script Parameters — DHCS LTC D-SNP (Both Brands)
All parameters declared at the top of each script. @BRAND is the sole architectural fork. Changes must be documented in the COMMENT(S) / CHANGE.LOG section with a C### reference per DUB C 2.0.
@BRAND
nvarchar(100) = 'CHGSD'
or 'CHPIV'
Execution fork. Drives @lobplanprog, output table names (DSNP vs CHPIV), and uvw_LINE_OF_BUSINESS BRAND filter. The single parameter separating the two brand scripts.
@leadlag
decimal(9,0) = -4
LAG months for @RangeStartDate. -4 = 4 months back from first of current month. Aligns reporting window to the appropriate quarterly submission period.
@leadlagmonths
int = 3
Window length in months from @RangeStartDate. 3 = one calendar quarter. @RangeEndDate = last day of 3rd month from @RangeStartDate.
@RangeStartDate
datetime2 = DATEADD(MONTH,
@leadlag,DATEADD(DAY,1,
EOMONTH(GETDATE(),-1)))
Dynamic. 1st of the quarter-start month. On 05/27/2026 with @leadlag=-4: = Jan 1, 2026.
@RangeEndDate
datetime2 = DATEADD(day,-1,
DATEADD(MONTH,@leadlagmonths,
@RangeStartDate))
Dynamic. Last day of 3-month window. On 05/27/2026: = Mar 31, 2026. Reporting Period = '2026 Q1'.
@CutoffDate
datetime2 = TRY_CONVERT(date,
GETDATE())
Paid date cutoff. TRY_CONVERT(date) = today. Applied as upper bound for paiddate filters where applicable.
@ltcdays
decimal(9,0) = 90
LTC membership threshold. WHERE DAY(S) >= 90 in DELIVERABLE build. Per DHCS requirement: count members in LTC for >90 days. Do NOT reduce below 90.
@los
decimal(9,0) = 30
Readmission window in days. READMISSION=1 where DoTheMathInDays BETWEEN 1 AND @los. 30-day readmission standard.
@gapallowance
int = 1
Consecutive stay gap tolerance in days. CONSECUTIVE=1 where DoTheMathInDays <= 1 AND RANKis != 1. Allows 1-day gap between claims to still count as one continuous stay.
@IndexPlus
int = 3
Consecutive + plus the IndexAdmit. Used in WHILE loop analysis context for index admit counting beyond the initial admit. Analyst reference parameter.
@lobplanprog
varchar(255) = 'DSNP'
or 'MSO - CHPIV'
LOB LIKE filter string. Applied as uvw_LINE_OF_BUSINESS.LINE_OF_BUSINESS LIKE '%'+@lobplanprog+'%'. Driven by @BRAND selection. SELECT from uvw_LINE_OF_BUSINESS to validate active options.
@memid
varchar(25) = NULL
QA isolation filter. NULL = all members. Set to a specific memid for member-level debugging. Applied in sample DETAIL SELECT (WHERE memid IN ('...')) — note: currently hardcoded sample in script.
