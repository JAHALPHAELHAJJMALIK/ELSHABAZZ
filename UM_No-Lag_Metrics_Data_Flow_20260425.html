Runbook Sequence
00 · SSA

Monthly SQL Server Agent job runs on day 28 at 04:01 AM.

01 · LTAC/SNF

Run NO_LAG AR_LTAC_SNF_REGULAR_...sql in SQLPROD02.

02 · HCT

Run NO_LAG_AR_HCT_...sql in SQLPROD02.

03 · SPD

Run NO_LAG_AR_SPD_...sql using HealthCareTrendReports root assets.

04 · JAG

Run JAG METRIC Readmission Logic...sql in SQLPROD02.

05 · Excel

Refresh HCT/LTAC/SNF pivot, SPD pivot, and JAG KPI workbooks.

06 · RAW/UI

Update RAW worksheet from refreshed HCT worksheets, then maintain the SharePoint UM UI.

Analyst Controls
ControlRecommended validationFailure symptom
Job completionConfirm SSA job success before dependent manual SQL execution.Downstream refresh uses stale or incomplete staging data.
Environment disciplineConfirm execution context is SQLPROD02 for all manual scripts.Workbook refresh points to inconsistent outputs.
File dependencyVerify HealthCareTrendReports root and RAR package availability.Missing source asset or outdated package reference.
Excel refreshRefresh all connections and pivots; confirm refresh timestamp or row counts where available.Pivot tables do not reflect latest no-lag run.
RAW reconciliationTrace RAW worksheet fields back to refreshed HCT data worksheets.UI values disagree with workbook or SQL-derived totals.
SharePoint governanceConfirm admin-only changes are routed through Ms. Jocelyn or authorized owner.List view / field configuration drift or permission blockage.
Senior analyst note: The workflow has one scheduled kickoff but remains a controlled manual production process. Strongest improvement opportunity is adding observable run-state fields: job completion timestamp, script execution timestamp, workbook refresh timestamp, RAW updated by, and SharePoint published by.
