ECM Continuous Population
Cost & Utilization Report
Identifies members continuously enrolled in Medi-Cal and ECM-active during an 18-month assessment window, then cross-references pre/post inpatient, outpatient facility, and professional claims to produce a PMPM utilization baseline. The COO-requested refresh spans CY2021 baseline vs. CY2025 measurement.

Dual-Track Design — Two Enrollment Populations
Track A — ECM Attribute Population (qattribute)
Anchors on HMOPROD_PLANDATA.dbo.memberattribute filtered by qa.description = 'ECM Enrolled' via CROSS APPLY. Reporting window: @ClockStartECM – @ClockStopECM (01/01/2022 – 06/30/2023). ISO calendar anchors to last day of month. Produces #RANKLAG → #ContinuousEnrollment → #TotalDaysEnrolled → #GAP. Members must have ≥ 12 continuous months. Output: #CONTINOUSENROLLMENT_FINAL.
Track B — Medi-Cal Enrollkeys Population (2021 Baseline)
Anchors on HMOPROD_PLANDATA.dbo.enrollkeys joined to uvw_LINE_OF_BUSINESS and intersected against #ECMEnrolled_FINAL. Reporting window: @CEClockStart – @CEClockStop (01/01/2021 – 12/31/2021). ISO day = 1st of month. segtype = 'INT', IsSubscriber = 'Y'. Produces #RANKLAG_v2 → #ContinuousEnrollment_v2 → #EligiblePopulation_v2. WHILE loop propagates consecutive index. Output feeds #CALAIM_Mem.
Analyst Note (WCARR): The dual-track design is intentional. Track A establishes WHO was ECM-enrolled for the 18-month window. Track B establishes WHO was continuously enrolled in Medi-Cal all of 2021. The intersection (#CALAIM_Mem = Track A ∩ Track B) is the analytical population for claim cost comparison. ECMEnrolled flag = 1 for 2022+ months, = 0 for 2021 baseline months.

Analyst Run Protocol — ECM Continuous Population Report
01 · Parameter Confirmation

Confirm all date parameters before execution. @ClockStartECM / @ClockStopECM set the 18-month ECM window (currently 01/01/2022 – 06/30/2023). @CEClockStart / @CEClockStop set the 2021 baseline (01/01/2021 – 12/31/2021). @ClockStartCostCategory / @ClockStopCostCategory set the measurement period (01/01/2025 – 12/31/2025). @BaselineClockStart/Stop must remain 2021. @lob = 'MEDI-CAL BENEFIT PLAN' — never NULL.

02 · #ECMEnrolled_FINAL Validation

Execute the ECMEnrolled_FINAL build block first. Run SELECT COUNT(DISTINCT memid) FROM #ECMEnrolled_FINAL to confirm the ECM roster is populated. Zero rows = memberattribute / qattribute connectivity issue or date fence misconfiguration. Do not proceed if zero.

03 · Track A Enrollment Run (#RANKLAG → #CONTINOUSENROLLMENT_FINAL)

Execute Track A blocks sequentially. After #CONTINOUSENROLLMENT_FINAL, validate: SELECT COUNT(DISTINCT memid) FROM #CONTINOUSENROLLMENT_FINAL. Expected to return the 2021 continuously enrolled cohort (Total Month(s) Enrolled = 12). If empty, check enrollkeys segtype = 'INT' and IsSubscriber = 'Y' filters, and verify @CEClockStart / @CEClockStop date fence is correct.

04 · Track B Enrollment Run (#RANKLAG_v2 → #EligiblePopulation_v2)

Execute Track B blocks. The WHILE loop on #ContinuousEnrollment_v2 runs until @@ROWCOUNT_v2 = 0 — this may take several seconds on large populations. After WHILE loop, validate #EligiblePopulation_v2 row count. All index propagation must complete before #CONTINOUSENROLLMENT_FINAL filter is applied.

05 · #CALAIM_Mem Intersection Validation

After both tracks complete, execute #CALAIM_Mem build. Run SELECT COUNT(DISTINCT memid) FROM #CALAIM_Mem. This is the final analytical population. If materially smaller than expected, investigate whether #ECMEnrolled_FINAL and #CONTINOUSENROLLMENT_FINAL overlap correctly. If zero, ALL downstream claim CTEs will return empty results silently.

06 · Claim CTE Execution (Inpatient → Outpatient → Professional)

Execute three claim CTE chains in order. After each INTO temp table, run a quick SELECT TOP 10 to confirm rows. Inpatient: verify paidfinal and days2 are non-null for at least some rows. Outpatient: verify admits_visits count. Professional: verify primarystatus logic is producing 'P' values. All three filter planid IN ('QMXBP0782','QMXBP0823','QMXBP0822') — confirm these planids are active.

07 · ##Base + UNION Assembly + SubCostCategory Rename

Execute ##Base build. Execute the three-way UNION INSERT into #QAPEERREVIEW_CalAimMemberClaims_Continuous_v3. Execute three UPDATE statements: ECMEnrolled=1 (all rows), ECMEnrolled=0 (2021), SubCostCategory rename CASE, Baseline Enrollment scalar stamp. Validate UPDATE row counts — SubCostCategory rename should touch all rows, Baseline Enrollment stamp should also touch all rows.

08 · Output Validation & Excel Delivery

Execute final SELECT * for member-level detail review. Execute aggregate summary SELECT (YEAR IN 2021, 2025, GROUP BY SubCostCategory). Validate: (a) Both years appear in aggregate output. (b) SubCostCategory covers all 9 expected labels. (c) Paid amounts and visit counts are non-zero for major categories. (d) Baseline Enrollment denominator is consistent across all rows. Forward aggregate summary to COO report workbook for refresh.

Peer Review Note (WCARR): The primary risk in this pipeline is the silent-zero pattern. If #ECMEnrolled_FINAL or #CONTINOUSENROLLMENT_FINAL return zero rows, the INNER JOIN producing #CALAIM_Mem returns zero, and all three claim CTEs LEFT JOIN against an empty table — producing zero-filled output with no error. The pipeline will complete successfully and return a "clean" output that is actually empty. Step 05 validation is the critical gate.

Cost Category Architecture — 3 Buckets · 9 Sub-Categories
Each claim form type feeds a dedicated CTE chain. The cost_category CASE logic is applied at claim extraction, then renamed to display labels via a final UPDATE statement. Output is consumed by the Excel COO report as a pre-aggregated YEAR × SubCostCategory pivot.
⬛ Inpatient — UB04
Seed: facilitycode IN ('1','2','8') AND billclasscode IN ('1','2') AND formtype = 'UB04'. Day-span splitting via INFORMATICS.Calendar with LAG boundary detection. paidfinal = paid × (days2 / days) weighted allocation. admits = ROW_NUMBER() = 1 per stay.
Inpatient Hospital Inpatient SNF Inpatient Other
CASE logic: facilitycode='1' + billclasscode IN 1/2 → Hospital; facilitycode='2' → SNF; else → Other. Original raw labels: 1-INPATIENT-HOSPITAL / 2-INPATIENT-SNF / 3-INPATIENT-OTHER.
⬛ Outpatient Facility — UB04
Seed: UB04 NOT matching inpatient criteria (NOT facilitycode IN 1/2/8 + billclasscode IN 1/2). visit_identifier = memid + provid + startdate concatenation. admits_visits = COUNT DISTINCT visit_identifier. No day-span splitting — single DOS aggregation.
Emergency Room Facility Non-ER
CASE logic: revcode LIKE '045%' AND facilitycode='1' AND billclasscode IN ('3','4') → '1-FACILITY-ER'; else → '2-FACILITY-OTHER'. Renamed: Emergency Room / Facility Non-ER.
⬛ Professional — 1500
Seed: formtype = '1500'. primarystatus logic: if stat1/stat2/stat3 all NULL or 'V' → treat as 'P'. Aggregates paid, admits_visits, days_units per memid / DOS_Month / cost_category. No day-span splitting.
Specialty Primary Care Emergency Transportation Other
CASE logic: 1-PROFESSIONAL-SPECIALTY → Specialty; 2-PROFESSIONAL-PCP → Primary Care; 3-PROFESSIONAL-EMERGTRANSPORT → Emergency Transportation; 4-PROFESSIONAL-OTHER → Other. Sub-category CASE logic (specialty vs PCP etc.) is in profwps CTE — confirm logic matches current billing patterns.
Final Output UNION: All three claim CTEs LEFT JOIN against ##Base (member × month × cost_category scaffold) to ensure zero-fill for months with no claims. The three result sets are UNIONed with consistent column order: MemID · ECMEnrolled · ECM_Provider_ID · ECM_Provider_Name · MemberCategory · MemberMonth · CostCategory · SubCostCategory · Paid · Admits_Visits · Days · carriermemid · QNXTLanguage · EICDesc · sex · [Baseline Enrollment].
Peer Review Flag (WCARR): The Professional CTE contains a SELECT * FROM #OutptMemberComparison on line ~1532 outside the CTE chain — this appears to be a debug/validation SELECT left in the production script. This will emit an additional result set during execution. Should be commented out or removed before COO delivery run. Additionally, #QAPEERREVIEW_CalAimMemberClaims_Continuous_v3 has a commented-out TRUNCATE/INSERT pattern at the top — if the table is intended to be permanent/persistent across runs, the TRUNCATE should be uncommented. Currently the script uses DROP TABLE IF EXISTS pattern implicitly through the INTO clause, which creates a new temp table each session.

date_calendarCE

40-year calendar 1st-of-month only. Track B date spine. calendar_day = @isoday (1). Used with CROSS APPLY on enrollkeys. Narrower coverage than ISO — confirm max date.

LOB metadata view joining enrollkeys to LINE_OF_BUSINESS string. Used in Track B to filter @lob = 'MEDI-CAL BENEFIT PLAN'. Also carries PROGRAM field.

ECM enrollment signal table. Joined to qattribute on attributeid. Track A anchor — CROSS APPLY against date_calendarISO. Contains effdate, termdate per attribute per member.

HMOPROD_PlanData — Claims
claim
Source
Claim header. Fields: claimid, memid, facilitycode, billclasscode, formtype, startdate, enddate, admitdate, cleandate, provid, affiliationid, planid, status, resubclaimid. Status filter: ADJUCATED/PAID/PAY/REV/REVERSED/REVSYNCH. Exclude resubclaimid + claimid LIKE '%R%'.
claimdetail
Source
Claim line detail. Fields: claimid, claimline, revcode, amountpaid, servunits. status filter: NOT IN ('DENY','VOID'). servunits < 100,000 for inpatient. revcode used for ER detection (LIKE '045%').
