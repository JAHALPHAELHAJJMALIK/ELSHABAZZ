Pre-Delivery QA Checklist — run before every contracting script submission
CheckMethodFailure Symptom
Rev code zero-padding SELECT DISTINCT revcode FROM [TABLE] WHERE revcode IN ('0120','0169','0200','0206') — confirm leading zero present WHEN branches never fire — all lines fall to ELSE. Proposed_Rate_Payment = 0 or PAID_NET_AMT uniformly.
HCT LOGIC DAY(S) NULLs GROUP BY revcode · SUM(CASE WHEN [HCT LOGIC DAY(S)] IS NULL THEN 1 ELSE 0 END) AS NULL_Day_Count — expected 0 for all Per Diem rev codes NULL output on Per Diem branches — proposed total understated. Analytics template shows NULL cells.
NULLIF PAID_NET_AMT check SELECT revcode, PAID_NET_AMT, NULLIF(ISNULL(PAID_NET_AMT,0),0) AS NULLIF_Check — confirm 0.00 lines collapse to NULL (expected behavior) Zero-dollar lines (e.g. rev 0942 Educ/Training) show $0.00 proposed — correct, not a bug when PAID_NET_AMT = 0.00.
LOB bleed-through SELECT DISTINCT LINE_OF_BUSINESS FROM #ContractAnalysis — confirm only expected LOBs present DSNP lines present in Medi-Cal-only script — ELSE branch applies flat PAID_NET_AMT, suppressing proposed uplift for those members.
@currentmultiplier DECLARE Confirm DECLARE @currentmultiplier AS decimal(9,2) and SET at top of script before CASE block Msg 137: must declare scalar variable. Script fails at runtime — no output produced.
CASE branch precedence Confirm Per Diem / DRG branches appear BEFORE LOB catch-all branches. SQL CASE evaluates top-to-bottom and stops at first match. Specific service lines (LTAC, Psych, Car-T) fall into LOB-level catch-all — rate calculation incorrect.
Counter_Offer_Payment SELECT MIN(Counter_Offer_Payment), MAX(Counter_Offer_Payment) — confirm no NULLs and values are ~5% above Current_Rate_Payment NULL Counter values signal EQUIV_allow_compare and PAID_NET_AMT both null on those lines — upstream data gap.
Analytics template reconciliation Compare Proposed_Rate_Total in script output vs. Provider Rate Increase Request Analytics Template (Excel). Variance > 0.5% warrants investigation. Multiplier error in CASE expression, DOS date boundary mismatch, or claims lag offset not matching Contracting's date range.
DUB C 2.0 compliance Visual scan: one space between all tokens, left-aligned commas, no bare string group expressions, explicit AS on all aliases, uppercase keywords Cosmetic only — but non-compliant code submitted to Sir Phil or Sir Salim will be returned for revision.
Senior Analyst Note (WCARR): The two highest-frequency failure modes in the six-provider corpus are (1) rev code zero-padding mismatches causing ELSE fallthrough, and (2) HCT LOGIC DAY(S) NULL values on Per Diem lines producing silent zero output. Both checks take under two minutes and eliminate the most common resubmission cycles. Run them before every delivery without exception.
