Report Purpose
IHA Medi-Cal Completion Tracking
Measures the rate at which newly enrolled Medi-Cal members (age ≥ 20, continuous enrollment ≥ 3 months) receive their Initial Health Assessment within each enrollment cohort quarter. IHA credit is sourced from four independent pathways: QNXT preventive visit claims, QNXT IHEBA claims, QNXT CHDP module (SQLPROD01), and DHCS Care Coordination files (CHGAPP_PROD / APCD). Source Tables Reference — All Tables Touched by sp_IHA_MEDI_CAL
Six databases / servers contribute to this report. Work tables are created and dropped each run (session-scoped). Permanent INFORMATICS tables are dropped and recreated each run. QNXT and external tables are read-only sources.
INFORMATICS · dbo — SQLPRODAPP01 (Analytics Warehouse)
dbo.IHA_MEDI_CAL
Final Output
Production output table. Written at the end of each run. Holds all eligible members, their IHA completion flags, completion dates, and PCP attribution. Source of the two summary result sets.
dbo.IHEBA
Work Table
IHEBA visit claim staging. Sourced from HMOPROD_PlanData.claim + claimdetail filtered to CPT codes 99420, 96160, 96156, 96150. Dropped and recreated each run.
dbo.MINiheba
Work Table
First IHEBA visit date per member. MIN(startdate) from IHEBA grouped by memid. Used to SET [IHEBA performed (Y/N)] = 'Y' on DaysEnrolled.
dbo.IHAclaimsPREVENTIVE
Work Table
Preventive visit claim staging. Sourced from HMOPROD_PlanData.claim + claimdetail filtered to E&M preventive CPT ranges. Specialty-filtered via #PCPSPE. Dropped and recreated each run.
dbo.MINiha
Work Table
First preventive visit date per member. MIN(startdate) from IHAclaimsPREVENTIVE. Used to SET [Preventive Visit exam performed (Y/N)] = 'Y'.
dbo.CHDP_IHA
Work Table
CHDP encounter staging. Sourced from SQLPROD01.QPROD.dbo.CHDP_CLAIM_HEADER. Field-based qualification: HISTORY_PHYS_EXAM or DEVELOPMENTAL_ASSESS not blank/B, PARTIAL_SCREEN='0'. PROCCDE = 'CHDP'.
dbo.MINchdp
Work Table
First CHDP encounter date per member. MIN(startdate) from CHDP_IHA. Updates both [Preventive Visit exam performed] and [IHEBA performed] flags on DaysEnrolled.
dbo.IHAdhcs
Work Table
DHCS care coordination staging. Sourced from CHGAPP_PROD.dbo.tblEDIDHCSCareCoordinationHeader (old format) and [CHGAPP_PROD].[APCD].[Medical] (new format eff. 7/1/22). 12-month pre-enrollment lookback.
dbo.MINdhcs
Work Table
First DHCS care coordination visit date per member. MIN(startdate) from IHAdhcs. Third pathway to update [Preventive Visit exam performed] = 'Y'.
dbo.DaysEnrolled
Work Table
The central working population table. Holds all eligible members passing LOB, age, continuous enrollment, newborn, and Medicare filters. Progressively updated with IHA flags from all four pathways. Written to IHA_MEDI_CAL at end of run.
dbo.MINmem / dbo.MAXmem
Work Table
Member's earliest effective date (MINmem) and latest termination date (MAXmem) in Medi-Cal. Used to define the enrollment window and remove members whose initial enrollment falls outside ClockStart–ClockStop.
dbo.DE / dbo.DEfinal
Work Table
Total enrolled days per member (DE) and final days-enrolled dataset (DEfinal) after all gap and continuity filters are applied. Joined to enrollkeys and MINmem to build DaysEnrolled.
dbo.uvw_LINE_OF_BUSINESS
View
Translates QNXT programid / planid into LINE_OF_BUSINESS, PLAN, and PROGRAM labels. The primary LOB filter used throughout: LINE_OF_BUSINESS LIKE '%MEDI-CAL%'.
dbo.uvw_PCP
View
Returns each member's most current PCP assignment. Internally ranks by effdate DESC using DENSE_RANK(). Filtered to @PCPrecordRANK = 1 in DaysEnrolled assembly.
dbo.date_calendarISO
Reference
50-year static ISO calendar. The backbone of the continuous enrollment calculation. Joined to eligibility spans to produce consecutive month counts and gap measurements.
HMOPROD_PlanData · QNXT Transactional (SQLPROD02)
dbo.claim
Source
Header-level claim records. Provides memid, claimid, planid, provid, startdate. Joined to claimdetail for CPT code access. Used for both IHEBA and preventive visit pathways.
dbo.claimdetail
Source
Line-level claim records. Provides servcode (CPT/HCPCS code). The CPT code filter for all QNXT-sourced IHA pathways is applied here.
dbo.enrollkeys
Source
Master enrollment roster. Provides effdate, termdate, programid, planid, carriermemid, memid, segtype. Program ID lock: QMXHPQ0842. Used in continuous enrollment logic, DaysEnrolled assembly, MINmem, MAXmem, and DE build.
dbo.member
Source
Member demographics. Provides DOB (for age calculation), primarylanguage (linked to language table), entityid, headofhouse (used as fallback for Health Plan ID when carriermemid is blank).
dbo.enrollstatus
Source
Enrollment status history. Provides primarystatus. Members where primarystatus ≠ 'P' are deleted from DaysEnrolled — excludes Medicare-primary members per Ms. Gabriela 20210604.
dbo.enrollcoverage
Source
Enrollment coverage details. Provides ratecode. Joined in the status indicator subquery alongside enrollstatus to surface rate code context for each member's active coverage segment.
dbo.memberrelation
Source
Member relationship records. Joined to qrelationship to identify newborn dependents. Relationship code C00498102 = dependent covered under mother. Matching BABYmemid records are flagged 'DELETE' and removed from DaysEnrolled.
dbo.qrelationship
Reference
Relationship code lookup table. Maps relationid to a description. Used to identify the 'DEPENDENT' relationship (C00498102) for the newborn exclusion logic.
dbo.language
Reference
Language code lookup. Maps member.primarylanguage (numeric ID) to a language description. LEFT JOINed in the final DaysEnrolled assembly for display purposes.
CHGAPP_PROD — DHCS Care Coordination & APCD (Linked Server)
dbo.tblEDIDHCSCareCoordinationHeader
Linked Server · Old Format
OLD format DHCS care coordination file (pre 7/1/2022). Keyed on CIN / MemId. Provides DateofServiceFrom for 12-month pre-enrollment IHA lookback. Staged into INFORMATICS.dbo.IHAdhcs.
[APCD].[Medical]
Linked Server · New Format
NEW format APCD (All-Payer Claims Database) medical claims table (effective 7/1/2022). Keyed on PatientControlNumber + CarrierSpecificUniqueMemberID. Provides DateofServiceFrom. Staged into the same IHAdhcs pipeline. Both old and new format run together so the 7/1/22 transition is seamlessly covered.
SQLPROD01.QPROD — CHDP Module (Linked Server)
dbo.CHDP_CLAIM_HEADER
Linked Server · Source
CHDP encounter records. Exclusively Medi-Cal (CL1). Qualification based on HISTORY_PHYS_EXAM or DEVELOPMENTAL_ASSESS fields (not blank, not 'B') AND PARTIAL_SCREEN = '0'. Staged into INFORMATICS.dbo.CHDP_IHA. Note in code: confirming with Mr. A. Hancock whether table is still actively updated.
TempDB — Session-Scoped Temp Tables (dropped end of run)
#PCPSPE
Temp Table
Provider specialty filter. Built from ProviderRepository. Holds provid, provider name, specialty code (FAM, GNP, INM, OBG, GYN, PED). Joined to claim on provid in both IHEBA and preventive visit queries to restrict which providers can generate IHA credit.
#ContinuousEnrollment
Temp Table
LAG/LEAD-enriched enrollment spans. Built from #RANKLAG. Holds GAPinDAYs, GAPinMONTHs, CONSECUTIVE fields per enrollment segment. The core continuous enrollment calculation happens here.
#TotalDaysEnrolled
Temp Table
Aggregated total days and months enrolled per member (SUM across all continuous segments). Also holds GAP(s)of count. Joined to enrollkeys to produce INFORMATICS.dbo.DE.
#RANKLAG
Temp Table
Intermediate ranking table for eligibility segments. Uses DENSE_RANK to sequence enrollment rows per member. Input to #ContinuousEnrollment. Dropped after #ContinuousEnrollment is built.

Senior Analyst Note (WCARR): The primary risk area introduced in v2 is the CHGAPP_PROD linked server dependency. Both the old-format (tblEDI) and new-format (APCD) tables must return rows or the DHCS/APCD pathway silently contributes zero credit — which can depress completion rates without an obvious error. Always validate MINdhcs row count after execution. The APCD format switchover on 7/1/22 means both tables must be queried together for claims spanning that boundary.
Stored Procedure Parameters — sp_IHA_MEDI_CAL
All parameters declared at the top of the SP. Changes must be documented in the COMMENT(S) / CHANGE.LOG section with a C### reference per DUB C 2.0.
@ClockStart
date = '01/01/2020'
Report start date. Per Ms. Gabby email 20211215. Anchors enrollment and IHA lookback.
@ClockStop
date = GETDATE()
Dynamic end date. Resolves to today. Ensures current-day eligibility cutoff each run.
@lob
nvarchar = 'MEDI-CAL'
LOB filter via LIKE on LINE_OF_BUSINESS. NULL = all LOBs — never appropriate for this report.
@ContinuousMonths
decimal = 3
Minimum consecutive enrollment months required. Do not reduce below 3.
@AgeCutoff
decimal = 20
Members must be ≥ 20 years old. Targets adult Medi-Cal population.
@GAPOF / @MAXGAP

Enrollment gap tolerance. A 1-month gap is allowed; gaps > 1 break the continuous window.
@lag / @lagyrs / @lagdays
int = 0 / 0 / 0
Date offset parameters. All zero = no lag. Adjust +/- for retrospective analysis only.
@PCPrecordRANK

@CCIIndicator

@isoday

ISO calendar day for effective date determination. Default = 1 (first of month).
@isomem
